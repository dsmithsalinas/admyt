# Admyt — Naming Conventions Guide

*How to name features so the brand stays warm and consistent without becoming confusing. The companion to the brand story and Sage personality guide.*

---

## The core principle

**Playful branded names are the default — but clarity always wins.**

Admyt's warmth comes partly from giving things real names instead of generic labels (Sage, not "AI Assistant"; Vibe Check, not "Culture Analysis"). That personality is an asset. But a branded name only works if a stressed 17-year-old instantly gets what it does. The moment a cute name creates confusion, we use the clear name instead.

**The test:** Say the name to someone who's never used Admyt. If they immediately understand *what it does* (or it's intriguing enough that they want to find out, and the payoff is obvious), it earns a branded name. If they'd be confused or have to guess, keep it descriptive.

---

## The three tiers

**Tier 1 — Branded (hero features that define the experience)**
These get full personality. They're the things that make Admyt *Admyt*. They're worth teaching users, because they become part of how people talk about the product.
- **Sage** — the AI advisor / friend
- **Vibe Check** — the campus culture fit analysis

**Tier 2 — Branded-but-clear (features that benefit from warmth and stay obvious)**
A light branded touch where the name still telegraphs function. Use when a playful name is genuinely clearer or warmer *and* unmistakable.
- Candidates: a match score, a saved-schools list, the student's profile of preferences
- Example directions (decide per feature): "Fit Score" (clear + branded), "My List" (warm + obvious), "Your Vibe" (the saved quiz profile)

**Tier 3 — Descriptive (utilities where clarity is everything)**
Plain language. These are tools people need to use correctly, often under stress and deadline pressure. Confusion here has real consequences (missed deadlines, etc.).
- **Application tracker** — stays descriptive. A cute name here risks someone not realizing it's where their deadlines live.
- **Reminders** — stays descriptive.
- Financial tools — stay clear (people need to trust and understand anything involving money).

---

## Feature-by-feature recommendations

**Sage** *(Tier 1 — locked)*
The AI friend. Core to the brand. Never "the assistant" or "the bot."

**Vibe Check** *(Tier 1 — locked)*
The signature feature. Owns the word "vibe" so the brand line doesn't have to.

**Match score** *(Tier 2 — consider "Fit Score")*
"Fit Score" ties directly to the brand promise ("find where you fit") and stays instantly clear. Recommended if you want a light brand touch. "Match score" is a fine fallback.

**Saved / hearted schools** *(Tier 2 — consider "My List" or "My Schools")*
"My Schools" (already in use on the profile) is warm and obvious. Good as-is. Avoid anything cutesy that hides the function.

**The preferences profile Sage builds** *(Tier 2 — consider "Your Vibe" or "What Sage knows")*
"What Sage knows" (already in use) is charming and clear — it reinforces the relationship. Keep it. If the saved Vibe Check quiz becomes its own thing, "Your Vibe" could name that specifically.

**Application tracker** *(Tier 3 — keep descriptive)*
This is a deadline-and-status tool people rely on under pressure. Clarity is non-negotiable. Keep "application tracker." (If you ever want a *light* brand warmth, a subtitle like "your application tracker, minus the chaos" gives personality without sacrificing the clear name.)

**Essay guidance** *(Tier 3 leaning Tier 2 — clear first, light brand optional)*
This one *could* carry a branded name since it's a standout feature, but only if it stays obvious. The risk: essay help is high-stakes and students need to trust it. Recommendation: lead with clarity ("essay help" / "essay guidance"), and if you want warmth, let Sage frame it in-conversation ("want help telling your story?") rather than inventing a name students have to decode. If a branded name emerges that's instantly clear *and* adds trust, revisit.

**Reminders** *(Tier 3 — keep descriptive)*
Functional, deadline-critical. Keep plain. Let the *voice* carry the brand — a reminder written in Sage's voice ("hey, that deadline's in 3 days 👀") is on-brand without needing a branded label.

**Share with friends** *(Tier 3 — keep descriptive)*
Self-explanatory and social. No need to brand it.

---

## Rules of thumb

1. **Hero experiences earn names. Utilities stay clear.** The more emotional and identity-defining a feature, the more it can carry a branded name. The more functional and consequence-heavy, the more it should just say what it does.

2. **Never sacrifice clarity for cuteness.** If a student could miss a deadline, misunderstand a cost, or fail to find a feature because of a clever name, the clever name loses.

3. **Let Sage's voice carry warmth where names can't.** A descriptively-named feature can still feel warm because Sage talks about it like a friend. The name is the label; the voice is the personality. You don't need both on every feature.

4. **Branded names should age well.** Like the "vibe" discussion — avoid names that lean on slang that could date. Sage and Vibe Check work because they're rooted in meaning, not trend.

5. **Consistency within a tier.** Don't have one branded feature and one generic one sitting side by side doing similar jobs — it reads as unfinished. Group similar things at the same level of polish.

6. **When in doubt, ship descriptive and brand it later.** It's easy to add personality to a clear name once a feature proves itself. It's painful to walk back a confusing branded name users have already learned.
